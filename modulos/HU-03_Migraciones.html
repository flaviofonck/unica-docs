    HU-03: Gestión de Migraciones y Ampliaciones¶

    Metadata¶

    Campo Valor
    Épica MÓDULO DE PROCESAMIENTO ÚNICA (2)
    Historia de Usuario 3. Gestión de Migraciones y ampliaciones
    Fecha de elaboración 05/12/2025
    Versión 2.0 (06/03/2026)
    Elaborado por Quality Customer Service (QCS)

    Contexto de Negocio¶

    Persona: Ingeniero Especialista UNICA

    Impacto en negocio: Consolidar y asegurar la calidad y completitud de la información de manera transversal para los procesos de Migración, Ampliación y Bajas.

    Captura de valor:

    • Línea base: 484 días de procesamiento manual al año

    • Meta: reducir 338 días (70%)

    • Tiempo estimado final: 145 días hábiles al año

    Historia de Usuario¶

    Como Ingeniero Especialista UNICA, Quiero disponer de un módulo que permita cargar de manera segura la gestión de consolidación de información y seguimientos de los archivos de Migración, Ampliación y Bajas, Para que el inventario se actualice de manera automática una vez finalizados los procesos.

    Criterios de Aceptación¶

    1. Cargue de Archivo de Migración¶

    1.1 Pestaña¶

    Se crea la pestaña "Migraciones y Ampliaciones" en la herramienta.

    1.2 Opción: "Cargar archivos de Migración"¶

    El usuario carga el archivo de Migraciones y Ampliaciones desde el front. El sistema crea una tabla con nombre:

    • Nombre del proyecto: asignado por el usuario ejecutor.

    • Fecha: fecha del cargue del archivo (formato DD-MM-AAAA).

    1.4 Exportar formulario¶

    Opción para exportar a Excel o CSV para entrega al CGP o líder del proyecto.

    2. Consolidar Información (Datos Comerciales)¶

    2.1 Opción: "Consolidar información de Migración"¶

    Al seleccionar esta opción, el sistema valida contra el inventario actual de servicios de UNICA y trae al formulario (usando CUS/DS como llave) los siguientes campos:

    Campo comercial
    ESTADO CLÁUSULA UK
    Duración contratada (meses)
    Fecha fin cláusula proveedor
    MRCACTUALFACTURACION
    BW FACTURACIÓN

    2.2 Datos de cotización¶

    También se trae del módulo de cotización (llave: CUS/DS) los siguientes campos:

    Campo Fuente
    MRC PRECIOS Cotización → Costo Mensual MRC
    NRC PRECIOS Cotización → Costo Instalación NRC
    MEDIO Cotización → Medio Proveedor
    TIEMPO DE ENTREGA Cotización → Fecha estimada entrega de servicio
    TIPO_DE_RED Vacío — diligenciar por usuario
    COMENTARIO Cotización → Observación

    Si no existe información, se permite diligenciar los campos manualmente.

    3. Consolidación de Información Técnica¶

    3.1 Opción: "Consolidar información Técnica"¶

    El usuario carga la tabla PROYECTO_FECHA_MIGRACIONES y el sistema actualiza el formulario (puntos 1 y 2) cruzando por CUS (DS) los siguientes campos técnicos:

    Campo técnico
    ÁREA RESPALDO
    CONTACTO CGP/ASEGURAMIENTO
    PROVEEDOR ACTUAL
    ID ACTUAL
    Contacto en Sitio / Correo / Celular
    Coordenadas
    PE ACTUAL / IP PE ACTUAL / INTERFACE ACTUAL
    VPRN / SAP ACTUAL / WAN EN PE / Loopback
    VLAN ACTUAL / ENRUTAMIENTO / RUTA
    BW ACTUAL
    Voz / Video
    PLATINO / ORO / PLATA / BRONCE
    OBSERVACIÓN DATOS SEDE
    OBSERVACIONES REVISIÓN ÁREA RESPALDO

    Si no existe información o cambió la estructura, se permite diligenciar manualmente.

    4. Completitud de Información y Generación de Inconsistencias¶

    4.1 Validación BW¶

    Al consolidar información técnica, el sistema compara:

    • BW ACTUAL (técnico, punto 3) vs. BW FACTURACIÓN (inventario, punto 2).

    4.2 Si hay diferencias:¶

    • Se genera una alerta visual.

    • Se genera un archivo de inconsistencias con nombre: Inconsistencias_facturación_PROYECTO_FECHA_MIGRACIONES.

    • El usuario puede corregir la información desde el formulario digital (puntos 1 y 2).

    4.3 Botón "Cerrar Migración"¶

    Permite al usuario ejecutor cerrar la data del proyecto una vez ajustada.

    5. Consolidación Información VLAN e Interconexión (Casos Especiales)¶

    5.1 Opción: "Descargar información para Implantación"¶

    Genera un archivo para solicitar información de VLAN e interconexión al área de implantación.

    Nombre del archivo: Implantación_PROYECTO_FECHA_MIGRACIONES

    Campo Fuente
    DS, OT TT, BW, CLIENTE FINAL, DIRECCIÓN, MUNICIPIO, Departamento, Proveedor, VLAN Actual Archivo PROYECTO_FECHA_MIGRACIONES
    ITX PE NEW Campo nuevo — diligenciar
    IP ITX PE NEW Campo nuevo — diligenciar
    PTO ITX NEW Campo nuevo — diligenciar
    VLAN ASIGNADA IMPLANTACIÓN Campo nuevo — diligenciar

    6. Seguimiento de Migraciones, Bajas y Ampliaciones¶

    Estructura del archivo de Migración (campos de seguimiento):

    Campo
    CUS (DS), CLIENTE, DIRECCIÓN, Ciudad, ID
    Proveedor más barato, OT MIGRACIÓN
    ESTADO ACTUAL, AVANCE
    NUEVO ID NUEVO PROVEEDOR
    CLÁUSULA DE PERMANENCIA
    FECHA MIGRACIÓN REALIZADA
    TAREA EN BAJA EN ETB, FECHA ENVÍO BAJA DE PARTE DE UNICA
    ÁREA RESPALDO, CONTACTO CGP/ASEGURAMIENTO
    PROVEEDOR ACTUAL, ID ACTUAL
    Contacto en Sitio / Correo / Celular
    Coordenadas, PE ACTUAL, IP PE ACTUAL
    INTERFACE ACTUAL, VPRN, SAP ACTUAL
    WAN EN PE, Loopback, VLAN ACTUAL
    ENRUTAMIENTO, RUTA, BW ACTUAL
    Voz, Video, PLATINO/ORO/PLATA/BRONCE
    OBSERVACIÓN DATOS SEDE

    Estructura del archivo de Bajas:

    Campo
    DS, Ciudad, Departamento, Cliente, Dirección
    Proveedor Actual, ID ACTUAL, BW Kbps
    DURACIÓN CONTRATO
    CONTACTO BAJA NOMBRE / CORREO / CELULAR
    FECHA ENVÍO BAJA DE PARTE DE UNICA

    Estructura del archivo de Ampliaciones:

    Campo
    COMERCIAL, A Kbps, DS, CLIENTE, SEDE
    FECHA AMPLIACIÓN, OT, TAREA, Observación
    LEGALIZADO, OT LEGALIZACIÓN
    TAREA LEGALIZACIÓN, FECHA LEGALIZACIÓN

    Los archivos se pueden cargar múltiples veces hasta cerrar el proyecto.

    7. Actualización Automática del Inventario¶

    Cuando se registre una fecha válida en los campos clave, el inventario se actualiza automáticamente:

    Proceso Campo disparador Actualización en inventario
    Migración FECHA MIGRACIÓN REALIZADA Proveedor, MRC, Medio, BW Actual
    Bajas FECHA ENVÍO BAJA DE PARTE DE UNICA Estado de la DS
    Ampliación FECHA AMPLIACIÓN BW ACTUAL, MRC

    8. Reporte Final de Migraciones¶

    8.1 Opción: "Reporte Final Migraciones"¶

    Genera un archivo final con nombre: DOCUMENTACIÓN_FINAL_PROYECTO

    Campos del reporte:

    Campo
    TAREA UNICA, Solicitud OT, FECHA DE ENVÍO TERCERO
    MEDIO, PERMANENCIA, NRC, MRC
    FECHA ESTIMADA DE ENTREGA DEL SERVICIO
    ID ACTUAL, DS, BW ACTUAL (Kbps), BW NUEVO (Kbps)
    TIPO DE MODIFICACIÓN, CLIENTE
    DIRECCIÓN INSTALACIÓN ANTIGUA / NUEVA
    MUNICIPIO, DEPARTAMENTO, PROVEEDOR
    OT PROVEEDOR, PEP, Fecha migración

    Notas Técnicas¶

    • La llave de cruce en todos los procesos es CUS (DS).

    • El inventario se actualiza automáticamente al registrar fechas válidas.

    • Los archivos de cargue siguen la convención: PROYECTO_FECHA_MIGRACIONES.

    • Todos los campos no encontrados permiten diligenciamiento manual por el usuario UNICA.
