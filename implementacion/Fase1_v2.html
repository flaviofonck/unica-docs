    Módulo 1 — HU-00: Acceso y Seguridad¶

    Descripción: Gestiona el login con JWT, administración de usuarios/perfiles/permisos y log de auditoría.

    Estado real (auditoría): 🟢 95% implementado. Brechas: sin refresh token (RN-38 parcial), log de auditoría solo registra login.

    1.1 — Login de usuario¶

    Pasos de prueba¶

    1. Navegar a http://localhost:4200/login.

    2. Ingresar un usuario con estado = ACTIVO y su contraseña.

    3. Hacer clic en Ingresar.

    Resultado esperado¶

    • ✅ El sistema autentica al usuario y redirige a /dashboard.

    • ✅ El menú lateral muestra únicamente los módulos con puede_ver = true del perfil del usuario (RN-02).

    • ✅ Se registra un log de acceso (usuario, fecha y hora) en log_acceso.

    • ✅ El token JWT tiene expiración de 8 horas (RN-38).

    Prueba de usuario inactivo¶

    1. Intentar iniciar sesión con un usuario con estado = INACTIVO.

    Resultado esperado¶

    • ✅ El sistema muestra mensaje de error: "Usuario inactivo. Contacte al administrador." (RN-01).

    • ✅ No se genera token ni acceso al sistema.

    Prueba de credenciales incorrectas¶

    1. Ingresar usuario o contraseña incorrectos.

    Resultado esperado¶

    • ✅ El sistema muestra mensaje de error de autenticación.

    • ✅ No se redirige al dashboard.

    ⚠️ Brecha conocida: El refresh token de 24h (RN-38) no está implementado. Los usuarios deben re-loguear cada 8 horas.

    1.2 — Gestión de Usuarios (solo Gerente/Líder)¶

    Pasos de prueba¶

    1. Iniciar sesión con perfil Gerente o Líder Profesional.

    2. Ir a Administración → Usuarios (http://localhost:4200/usuarios).

    3. Hacer clic en Crear Usuario.

    4. Completar: Nombre completo, Correo corporativo, Rol, Cargo, Estado.

    5. Guardar.

    Resultado esperado¶

    • ✅ El usuario queda creado con estado = ACTIVO.

    • ✅ Se registra log de auditoría con fecha y hora de creación.

    • ✅ El nuevo usuario puede iniciar sesión.

    Prueba de modificación¶

    1. Editar un usuario existente: modificar Cargo o Rol.

    2. Guardar.

    Resultado esperado¶

    • ✅ Los cambios se reflejan inmediatamente.

    • ✅ Se actualiza el log de auditoría.

    Prueba de inactivación¶

    1. Cambiar estado de un usuario a INACTIVO y guardar.

    2. Intentar iniciar sesión con ese usuario.

    Resultado esperado¶

    • ✅ El usuario queda INACTIVO.

    • ✅ El acceso queda bloqueado (RN-01).

    Prueba de restricción de perfil (RN-04)¶

    1. Iniciar sesión con un perfil Analista.

    2. Intentar acceder a http://localhost:4200/usuarios.

    Resultado esperado¶

    • ✅ El sistema muestra error de acceso denegado.

    • ✅ La opción de gestión de usuarios no aparece en el menú lateral.

    1.3 — Gestión de Perfiles y Permisos¶

    Pasos de prueba¶

    1. Ir a Administración → Perfiles (http://localhost:4200/perfiles).

    2. Seleccionar un perfil existente (ej. Analista Conciliación Terceros).

    3. Revisar los permisos asignados por módulo (puede_ver, puede_editar, puede_admin).

    4. Modificar un permiso y guardar.

    Resultado esperado¶

    • ✅ Los permisos quedan actualizados en base de datos.

    • ✅ Un usuario con ese perfil ve el cambio reflejado en su menú al siguiente login.

    • ✅ Si puede_ver = false para un módulo, ese ítem no aparece en el sidebar del usuario.

    1.4 — Brechas conocidas HU-00¶

    Brecha Severidad Descripción
    Refresh token 24h 🟡 Importante RN-38 parcial — JWT solo de 8h, sin refresh. Pendiente: endpoint /api/auth/refresh
    Log de auditoría incompleto 🟡 Importante Solo se registra login. Falta: logout, cambios de perfil, creación de usuarios
    Validación de contraseña fuerte 🟢 Menor Frontend no valida 8+ chars, mayúsculas, números

    Módulo 2 — HU-05: Inventario de Servicios¶

    Descripción: Repositorio centralizado de servicios terceros y catálogos del sistema.

    Estado real (auditoría): 🟢 90% implementado. CRUD operativo, todas las validaciones RN activas. Brechas son mejoras.

    2.1 — Inventario Servicios Terceros¶

    1. Ir a Inventario de Servicios → Inventario Servicios Terceros (http://localhost:4200/servicios).

    2. Verificar que carga la tabla dim_configuracion_servicios.

    3. Usar filtros disponibles (proveedor, estado, ciudad, etc.).

    Resultado esperado¶

    • ✅ La tabla carga con los registros vigentes.

    • ✅ Los filtros reducen correctamente el resultado.

    • ✅ Desplazamiento horizontal disponible cuando hay muchas columnas.

    • ✅ Indicador visual de carga mientras se traen los datos.

    Prueba de nueva configuración de servicio¶

    1. Hacer clic en Nuevo Servicio.

    2. Completar campos obligatorios, incluyendo NIT Cliente (10 dígitos, no inicia en 0 — RN-07).

    3. Si PEP inicia con E-1167, verificar que imputacion se llena automáticamente como 'L' (RN-09).

    4. Guardar.

    Resultado esperado¶

    • ✅ El registro queda guardado con estado activo.

    • ✅ NIT inválido (no 10 dígitos, o inicia en 0) → el sistema muestra error de validación (RN-07).

    • ✅ PEP con E-1167 → campo imputacion = 'L' auto-calculado (RN-09).

    Prueba de DS duplicado (RN-06)¶

    1. Intentar crear una segunda configuración activa para un DS que ya tiene una activa.

    Resultado esperado¶

    • ✅ El sistema bloquea la operación (RN-06).

    Prueba con campo moneda USD (RN-28)¶

    1. Crear servicio con moneda_mrc = 'USD'.

    2. Verificar que MRC_COP = MRC × TRM del mes se calcula automáticamente.

    Resultado esperado¶

    • ✅ El sistema convierte automáticamente USD → COP usando TRM del mes (RN-28).

    2.2 — Catálogos y Parámetros¶

    1. Ir a Inventario de Servicios → Catálogos y Parámetros (http://localhost:4200/configuracion).

    Dim Proveedores¶

    • ✅ Lista NIT, nombre corto, nombre SAP, estado, correos de contacto.

    • ✅ CRUD completo (crear, editar, inactivar) con confirmación antes de guardar.

    • ✅ NIT: 10 dígitos, no inicia en 0.

    Dim TRM¶

    • ✅ Lista valores TRM por mes con valor USD → COP.

    • ✅ Solo 1 TRM por año+mes — sistema bloquea duplicado (RN-12).

    • ✅ Error visible al intentar ingresar segundo registro para el mismo mes.

    Dim Clientes¶

    • ✅ Lista clientes con NIT, nombre corto, nombre SAP, estado.

    • ✅ NIT: 10 dígitos, no inicia en 0.

    Dim Contratos¶

    • ✅ Lista contratos con proveedor, condición de pago y facturación.

    • ✅ Al modificar condición de pago/facturación → actualiza dim_configuracion_servicios.

    Dim Tipo Servicio / Clasificación / Ciudades / Medios / Cobertura / Servicios¶

    • ✅ Cada tabla muestra contenido tabular y permite edición con confirmación.

    Prueba de autocomplete en campos cliente/proveedor¶

    1. En formulario de servicio, escribir parcialmente nombre o NIT de cliente.

    2. Verificar que aparece el componente AutocompleteSelectComponent.

    Resultado esperado¶

    • ✅ Aparecen sugerencias filtradas por nombre o NIT.

    • ✅ Al seleccionar, se auto-rellena el NIT y el nombre.

    • ✅ Se muestra el NIT confirmado debajo del campo como texto de solo lectura.

    2.3 — Importación de proveedores/clientes por Excel/CSV¶

    1. Desde la lista de Proveedores, hacer clic en Importar.

    2. Seleccionar archivo CSV o XLSX.

    Resultado esperado¶

    • ✅ Al finalizar: resumen de insertados, duplicados, errores.

    • ✅ Modal de detalle con las primeras 200 filas.

    • ✅ Ambos formatos .csv y .xlsx son aceptados.

    Estado real (auditoría): 🟢 95% implementado. Todas las RN operativas. Una brecha de integración con HU-06.

    1. Ir a Novedades y Prefactura → Indisponibilidades (http://localhost:4200/novedades/indisponibilidades).

    2. Verificar que el menú lateral muestra exactamente estas 5 opciones:

    3. Cargar archivo base de Incidencias
    4. Validación de ANS y exclusiones
    5. Generación de archivos de conciliación
    6. Cargue de respuestas de proveedores

    Resultado esperado¶

    • ✅ Las 5 opciones están visibles y navegan sin error 404.

    3.2 — Cargar archivo base de Incidencias¶

    Especificaciones del archivo Remedy¶

    Campo Valor
    Nombre archivo Closed TERCEROS Preview_MM_AAAA.xlsx
    Hoja BASE GENERAL
    Columnas 12 (columnas 10 y 11 comparten nombre — parsear por índice)
    Volumen ~20.000 a 30.000 registros
    Llave de cruce ID ENLACE (columna índice 9, base 1)
    1. Seleccionar el archivo XLSX y hacer clic en Cargar.

    Resultado esperado¶

    • ✅ Barra de progreso aparece inmediatamente.

    • ✅ El progreso avanza de 0% → 100% sin bloquear la UI.

    • ✅ Al completar, el modal de resultado se abre automáticamente.

    • ✅ Resumen: procesados, insertados, duplicados, errores.

    • ✅ Botón Descargar CSV completo disponible.

    Prueba de bloqueo de carga duplicada (RN-25)¶

    1. Intentar cargar el mismo archivo por segunda vez para el mismo mes.

    Resultado esperado¶

    • ✅ Sistema rechaza con mensaje: "Ya existe una carga de incidencias para el mes [Mes/Año]."

    • ✅ No se insertan registros duplicados.

    3.3 — Validación de ANS, exclusiones y motivos de fallas¶

    1. Ir a Validación de ANS y exclusiones.

    Tabla dim_ans_proveedores¶

    • ✅ CRUD activo para configurar umbrales de disponibilidad comprometidos por proveedor.

    • ✅ Los cambios impactan el cálculo de descuentos.

    Tabla dim_motivos_fallas — RN-24¶

    • ✅ Si motivo_fallo del ticket está en esta tabla → pct_descuento_final = 0.

    • Prueba: Agregar un motivo de falla. Recalcular. Verificar que los tickets con ese motivo quedan con descuento 0.

    Tabla dim_exclusiones_remedy¶

    • ✅ Patrones de exclusión: FRACTALIA-PDTI, MAXIMO, UNIX, WALTER BRIDGE, YEPDATA.

    • Prueba: Agregar patrón nuevo. Recargar incidencias. Verificar tickets descartados.

    3.4 — Filtros de validación aplicados durante la carga¶

    Filtro Condición Estado resultante
    Filtro 1 DS cruza con dim_configuracion_servicios Y fecha válida "Registro coincidente DS"
    Filtro 2 Vendor Ticket contiene patrón en exclusiones "Registro Descartado Remedy-Vendor" (no sobrescribe si ya es "coincidente")
    Filtro 2 Vendor Ticket NO contiene patrón exclusión "Registro identificado Remedy-Vendor"

    Prueba de fórmula RN-22:

    pct_disponibilidad = (((dias_mes × 1440) - indisp_minutos) / (dias_mes × 1440)) × 100

    1. Tomar un DS conocido.

    2. Sumar manualmente los minutos de los tickets del mes.

    3. Aplicar la fórmula.

    4. Comparar con el valor mostrado en pantalla.

    5. ✅ Diferencia ≤ 1 minuto (margen de redondeo).

    3.7 — Generación del consolidado mensual¶

    1. Ejecutar la generación del consolidado mensual.

    Resultado esperado¶

    • ✅ Tabla fact_indisponibilidad_mensual_conciliada se actualiza.

    • ✅ El archivo Excel del consolidado es descargable.

    • ✅ El módulo HU-08 puede ver el consolidado del mes en su validación de prerequisitos.

    Módulo 4 — HU-08: Prefacturación¶

    Descripción: Consolida ingresos Davox, genera prefacturas mensuales y valida contra ingresos reales.

    Estado real (auditoría): 🟢 90% implementado. RN-29 y RN-30 operativas. Brechas: export SAP y flujo aprobación.

    4.1 — Estructura del módulo (menú lateral — 3 opciones)¶

    1. Ir a Novedades y Prefactura → Prefacturación (http://localhost:4200/novedades/prefacturacion).

    2. Verificar que el menú lateral muestra:

    3. Cargar Ingresos Davox
    4. Generación de Prefacturas
    5. Validación Pagos vs Ingresos

    Resultado esperado¶

    • ✅ Las 3 opciones están visibles y navegan sin error 404.

    4.2 — Prerequisitos para generación de prefacturas (RN-29)¶

    Antes de poder generar prefacturas, el sistema debe verificar:

    1. Prorrateo de novedades del mes ejecutado (HU-06).

    2. Conciliación de indisponibilidades del mes ejecutada (HU-07).

    Prueba:

    1. Intentar generar prefacturas para un mes sin prorrateo ejecutado.

    Resultado esperado¶

    • ✅ El sistema bloquea con mensaje: "Debe ejecutar el prorrateo del mes antes de generar prefacturas." (RN-29).

    4.3 — Cargar Ingresos Davox (archivo FISC)¶

    1. Seleccionar Cargar Ingresos Davox.

    2. Cargar el archivo FISC (puede ser grande, >100MB).

    Resultado esperado¶

    • ✅ La importación corre en background asíncrono con barra de progreso.

    • ✅ Al finalizar: resumen de insertados, duplicados, errores.

    • ✅ Modal de detalle disponible con primeras 200 filas.

    • ✅ Botón Descargar CSV con resultado completo.

    • ✅ Acepta .csv y .xlsx.

    4.4 — Generación de Prefacturas¶

    1. Verificar que los prerequisitos del mes están completos (prorrateo + conciliación).

    2. Hacer clic en Generar Prefacturas del mes.

    Resultado esperado¶

    • ✅ Se genera la prefactura consolidada por proveedor:
    • MRC base + prorrateo novedades - descuento indisponibilidades = Total antes de IVA

    • IVA = Total_antes_IVA × indicador_iva (RN-30)

    • ✅ Conversión USD→COP con TRM del mes (RN-28).

    • ✅ Los resultados se guardan en fact_prefactura.

    • ✅ Tabla de prefacturas generadas con detalle por servicio descargable.

    4.5 — Validación Pagos vs Ingresos Davox¶

    1. Ir a Validación Pagos vs Ingresos.

    2. Seleccionar el mes y ejecutar validación.

    Resultado esperado¶

    • ✅ Tabla con columnas: DS, Proveedor, Prefactura, Ingreso Davox, Diferencia, % Diferencia, Estado.

    • ✅ Badge de estado:

    • 🟢 COINCIDE — diferencia ≤ 2%
    • 🟡 TOLERANCIA — diferencia entre 2% y 5%

    • 🔴 DISCREPANCIA — diferencia > 5%

    • ✅ Filtro por estado (mostrar solo DISCREPANCIA, etc.).

    • ✅ Exportar CSV / Excel con el reporte de validación.

    4.6 — Brechas conocidas HU-08¶

    Brecha Severidad Descripción
    Export a formato SAP 🟡 Importante Pendiente definir estructura exacta con QCS. Endpoint /api/prefactura/export-sap/{mes} no implementado
    Flujo de aprobación de prefactura 🟡 Importante No existe campo estado (GENERADA/APROBADA/RECHAZADA). Solo Gerente/Líder puede aprobar (RN-04)
    Dashboard métricas de prefacturación 🟢 Menor Sin totales de MRC, descuentos y % diferencia promedio
    Notificación discrepancias > 5% 🟢 Menor Sin alerta por email automática

    Flujo E2E Fase 1 — Validación completa¶

    1. Login (HU-00) → Dashboard

2. Inventario (HU-05) → Verificar servicios activos

3. Indisponibilidades (HU-07) → Cargar tickets → Conciliar → Generar consolidado

4. Novedades/Prorrateo (HU-06, prerequisito HU-08) → Ejecutar prorrateo del mes

5. Prefacturación (HU-08) → Cargar Davox → Generar prefactura → Validar pagos

    Orden crítico: HU-07 debe estar conciliado Y HU-06 ejecutado antes de correr HU-08 (RN-29).
