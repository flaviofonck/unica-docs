    TASK-004: module.guard + secciones[] en AuthController y frontend¶

    Estado: ✅ Completada — 24/03/2026
    Contexto: HU-00 §3c / §2b

    Backend AuthController — agrega secciones[] al LoginResponse: 

    private List<String> obtenerSeccionesVisibles(Usuario usuario) {
    return usuarioPerfilRepository.findByUsuario(usuario).stream()
        .flatMap(up -> permisoRepository.findByPerfil(up.getPerfil()).stream())
        .filter(p -> Boolean.TRUE.equals(p.getPuedeVer()))
        .map(Permiso::getSeccion)
        .filter(s -> s != null && !s.isBlank())
        .distinct()
        .toList();
}

    Correcciones de encoding (V24 + V25)¶

    V24 — fix encoding y rebuild permisos¶

    Estado: ✅ Aplicada — 25/03/2026
    Archivo: src/main/resources/db/migration/V24__fix_permisos_rebuild_encoding.sql

    Causa raíz: V6 y V9 insertaron permisos sin SET NAMES utf8mb4 → 22 variantes corruptas de (modulo, seccion) → 107 filas infladas.

    Pasos:

    1. Elimina UNIQUE constraint existente (condicionalmente)

    2. DELETE FROM permisos — elimina todos los registros corruptos

    3. Re-inserta 37 registros canónicos con SET NAMES utf8mb4

    4. Restaura UNIQUE (id_perfil, modulo, seccion)

    Resultado real: Solo insertó 23 permisos (perfiles id=2 e id=3 tenían nombres corruptos). Corregido por V25.

    V25 — fix encoding en perfiles + 14 permisos faltantes¶

    Archivo: src/main/resources/db/migration/V25__fix_perfiles_encoding_permisos_faltantes.sql

    1. UPDATE perfiles SET nombre_perfil = 'Líder...' WHERE id = 2 — corrige por id
