    Guía de Pruebas — Fase 1 (MVP F1)¶

    Sistema PROCESOS ÚNICA · Telefónica Colombia¶

    HU-00 · HU-05 · HU-07 · HU-08¶

    Fecha: 13/04/2026
    Preparado por: Equipo de Desarrollo QCS
    Versión documento: 1.0

    Tipo: Guía de Pruebas
    Fase: Fase 1
    Sprint: Sprint 1, Sprint 2
    Roles: QA, Cliente, PM
    Estado: 🟢 Activo
    HUs relacionadas: HU-00 · HU-05

    🕸️ Mapa de relaciones

    Acceso al sistema
    - Frontend: http://localhost:4200
    - Usuario de prueba: admin / admin (o el que corresponda al entorno)
    - Backend API: http://localhost:8080
    - Prerequisito: El backend debe estar corriendo (iniciar-backend.bat) y MySQL 8.4 activo.

    Módulo 1 — HU-00: Front de Acceso (Acceso / Seguridad)¶

    Descripción: Gestiona el login con JWT, administración de usuarios/perfiles/permisos y log de auditoría. Solo usuarios con estado = ACTIVO pueden iniciar sesión (RN-01). Solo perfiles Gerente/Líder Profesional pueden administrar usuarios (RN-04).

    1.1 — Login de usuario¶

    Pasos de prueba¶

    1. Navegar a http://localhost:4200/login.

    2. Ingresar un usuario con estado = ACTIVO y su contraseña.

    3. Hacer clic en Ingresar.

    Resultado esperado¶

    • ✅ El sistema autentica al usuario y redirige a /dashboard.

    • ✅ El menú lateral muestra únicamente los módulos para los que el perfil tiene puede_ver = true (RN-02).

    • ✅ Se registra un log de acceso (usuario, fecha y hora) en log_acceso.

    • ✅ El token JWT tiene expiración de 8 horas (RN-38).

    Prueba de usuario inactivo¶

    1. Intentar iniciar sesión con un usuario con estado = INACTIVO.

    Resultado esperado¶

    • ✅ El sistema muestra mensaje de error: "Usuario inactivo. Contacte al administrador."

    • ✅ No se genera token ni acceso al sistema.

    Prueba de credenciales incorrectas¶

    1. Ingresar usuario o contraseña incorrectos.

    Resultado esperado¶

    • ✅ El sistema muestra mensaje de error de autenticación.

    • ✅ No se redirige al dashboard.

    1.2 — Gestión de Usuarios¶

    Pasos de prueba¶

    1. Iniciar sesión con perfil Gerente o Líder Profesional.

    2. Ir a Administración → Usuarios (http://localhost:4200/usuarios).

    3. Hacer clic en Crear Usuario.

    4. Completar: Nombre completo, Correo corporativo, Rol, Cargo, Estado.

    5. Guardar.

    Resultado esperado¶

    • ✅ El usuario queda creado con estado = ACTIVO.

    • ✅ Se registra log de auditoría con fecha y hora de creación.

    • ✅ El nuevo usuario puede iniciar sesión.

    Prueba de modificación¶

    1. Editar un usuario existente: modificar Cargo o Rol.

    2. Guardar.

    Resultado esperado¶

    • ✅ Los cambios se reflejan inmediatamente.

    • ✅ Se actualiza el log de auditoría.

    Prueba de inactivación¶

    1. Cambiar estado de un usuario a INACTIVO y guardar.

    2. Intentar iniciar sesión con ese usuario.

    Resultado esperado¶

    • ✅ El usuario queda INACTIVO.

    • ✅ El acceso queda bloqueado (RN-01).

    Prueba de restricción de perfil¶

    1. Iniciar sesión con un perfil Analista.

    2. Intentar acceder a http://localhost:4200/usuarios.

    Resultado esperado¶

    • ✅ El sistema muestra error de acceso denegado.

    • ✅ La opción de gestión de usuarios no aparece en el menú.

    1.3 — Gestión de Perfiles y Permisos¶

    Pasos de prueba¶

    1. Ir a Administración → Perfiles (http://localhost:4200/perfiles).

    2. Seleccionar un perfil existente (ej. Analista Conciliación Terceros).

    3. Revisar los permisos asignados por módulo (puede_ver, puede_editar, puede_admin).

    4. Modificar un permiso y guardar.

    Resultado esperado¶

    • ✅ Los permisos quedan actualizados en base de datos.

    • ✅ Un usuario con ese perfil ve el cambio reflejado en su menú al siguiente login.

    • ✅ Si puede_ver = false para un módulo, ese ítem no aparece en el sidebar del usuario.

    Módulo 2 — HU-05: Inventario de Servicios¶

    Descripción: Repositorio centralizado con el inventario de servicios terceros y las tablas paramétricas del sistema. Accesible desde la sección INVENTARIO DE SERVICIOS del menú lateral.

    2.1 — Inventario Servicios Terceros¶

    Pasos de prueba¶

    1. Ir a Inventario de Servicios → Inventario Servicios Terceros (http://localhost:4200/servicios).

    2. Verificar que carga la tabla dim_configuracion_servicios.

    3. Usar filtros disponibles (proveedor, estado, ciudad, etc.).

    Resultado esperado¶

    • ✅ La tabla carga con los registros vigentes.

    • ✅ Los filtros reducen correctamente el resultado.

    • ✅ Desplazamiento horizontal disponible cuando hay muchas columnas.

    • ✅ Indicador visual de carga mientras se traen los datos.

    Prueba de nueva configuración de servicio¶

    1. Hacer clic en Nuevo Servicio.

    2. Completar campos obligatorios, incluyendo NIT Cliente (10 dígitos, no inicia en 0 — RN-07).

    3. Si PEP inicia con E-1167, verificar que imputacion se llena automáticamente como 'L' (RN-09).

    4. Guardar.

    Resultado esperado¶

    • ✅ El registro queda guardado con estado activo.

    • ✅ NIT inválido → el sistema muestra error de validación.

    • ✅ PEP con E-1167 → campo imputacion = 'L' auto-calculado (RN-09).

    Prueba de DS duplicado (RN-06)¶

    1. Intentar crear una segunda configuración activa para un DS que ya tiene una activa.

    Resultado esperado¶

    • ✅ El sistema bloquea la operación (RN-06).

    2.2 — Catálogos y Parámetros¶

    Pasos de prueba¶

    1. Ir a Inventario de Servicios → Catálogos y Parámetros (http://localhost:4200/configuracion).

    2. Seleccionar cada tabla del menú lateral y validar su funcionamiento.

    Resultado esperado por tabla¶

    Dim Proveedores:

    • ✅ Lista NIT, nombre corto, nombre SAP, estado, correos de contacto.

    • ✅ CRUD completo (crear, editar, inactivar) con confirmación antes de guardar.

    • ✅ NIT: 10 dígitos, no inicia en 0.

    Dim TRM:

    • ✅ Lista valores TRM por mes con valor USD → COP.

    • ✅ Solo 1 TRM por año+mes — sistema bloquea duplicado (RN-12).

    Dim Clientes:

    • ✅ Lista clientes con NIT, nombre corto, nombre SAP, estado.

    • ✅ NIT: 10 dígitos, no inicia en 0.

    Dim Contratos:

    • ✅ Lista contratos con proveedor, condición de pago y facturación.

    • ✅ Al modificar condición de pago/facturación → actualiza dim_configuracion_servicios.

    Dim Tipo Servicio / Clasificación / Ciudades / Medios / Cobertura / Servicios:

    • ✅ Cada tabla muestra contenido tabular y permite edición con confirmación.

    Módulo 3 — HU-07: Cálculo de Indisponibilidades¶

    Descripción: Carga el archivo de incidencias Remedy, valida ANS y exclusiones, genera archivos de conciliación para proveedores, registra respuestas y genera el consolidado mensual. Ruta: Novedades y Prefactura → Indisponibilidades (http://localhost:4200/novedades/indisponibilidades).

    3.1 — Cargar Archivo Base de Incidencias¶

    Pasos de prueba¶

    1. Ir a http://localhost:4200/novedades/indisponibilidades.

    2. Seleccionar "Cargar archivo base de Incidencias".

    3. Seleccionar el archivo Closed_TERCEROS_Preview_<mes>.csv o .xlsx.

    4. Hacer clic en Importar.

    Resultado esperado¶

    • ✅ Se inicia importación asíncrona con barra de progreso.

    • ✅ Se procesan: Incident ID, Vendor Ticket Number, Numero DS, fechas Reported/Closed, Customer Company/City/Site, Classification Service CI, ID ENLACE, Resolution Categories, Assignee Group.

    • ✅ Al finalizar: resumen con procesados, insertados, duplicados, errores.

    • ✅ Disponible descarga de CSV de resultado por fila.

    Prueba de carga duplicada (RN-25)¶

    1. Intentar cargar el archivo de un mes que ya fue cargado.

    Resultado esperado¶

    • ✅ Sistema bloquea la carga con mensaje: "Ya existe una carga de incidencias para el mes [Mes/Año]." (RN-25).

    3.2 — Validación de ANS y Exclusiones¶

    Pasos de prueba¶

    1. Una vez cargado el archivo, ir a "Validación de ANS y exclusiones".

    2. Seleccionar el mes procesado y ejecutar.

    Resultado esperado¶

    • ✅ Cruza tickets con dim_ans_proveedores para calcular minutos de indisponibilidad por servicio.

    • ✅ Tickets con motivo_fallo en dim_motivos_fallas → pct_descuento_final = 0 (RN-24).

    • ✅ Tickets aplicables a dim_exclusiones_remedy → descuento = 0.

    • ✅ Cálculo de % Disponibilidad: ((días_mes × 1440 − indisp_minutos) / (días_mes × 1440)) × 100 (RN-22).

    3.3 — Generación de Archivos de Conciliación¶

    Pasos de prueba¶

    1. Ir a "Generación de archivos de conciliación".

    2. Seleccionar el mes y generar para uno o todos los proveedores.

    Resultado esperado¶

    • ✅ Se generan archivos de conciliación por proveedor (CSV descargable).

    • ✅ Contienen tickets validados con descuentos calculados.

    3.4 — Cargue de Respuestas de Proveedores¶

    Pasos de prueba¶

    1. Ir a "Cargue de respuestas de proveedores".

    2. Cargar el archivo de respuesta del proveedor para el mes conciliado.

    Resultado esperado¶

    • ✅ Respuestas (aceptadas/rechazadas) registradas en fact_tickets_indisponibilidad.

    • ✅ Tickets rechazados marcados para revisión.

    • ✅ Resumen de respuestas procesadas visible.

    3.5 — Generación de Consolidado de Conciliación Mensual¶

    Pasos de prueba¶

    1. Ir a "Generación de consolidado de conciliación mensual".

    2. Seleccionar el mes y ejecutar.

    Resultado esperado¶

    • ✅ Registro generado en fact_indisponibilidad_mensual_conciliada con valor total a descontar por servicio y proveedor.

    • ✅ Resumen visible: proveedor, total minutos, % disponibilidad, valor descuento COP.

    • ✅ Consolidado disponible para consumo por Prefacturación (HU-08).

    Módulo 4 — HU-08: Prefacturación¶

    Descripción: Genera las prefacturas mensuales consolidando MRC, indisponibilidades conciliadas y mantenimientos. Requiere que el prorrateo de novedades y el consolidado de indisponibilidades del mes estén ejecutados (RN-29). Ruta: Novedades y Prefactura → Prefacturación (http://localhost:4200/novedades/prefacturacion).

    4.1 — Cargar Ingresos Davox (archivo FISC)¶

    Pasos de prueba¶

    1. Ir a http://localhost:4200/novedades/prefacturacion.

    2. Seleccionar "Cargar Ingresos Davox".

    3. Cargar el archivo FISC comprimido (.zip con .txt interno) del mes inmediatamente anterior.

    4. Hacer clic en Procesar.

    Resultado esperado¶

    • ✅ El sistema descomprime y procesa el archivo FISC (≈ 30 MB comprimido, ≈ 700 MB descomprimido).

    • ✅ Se filtran registros con ESTADO CUS = ACTIVO.

    • ✅ Cada registro se cruza con dim_configuracion_servicios por campo CUS/DS para identificar configuración vigente.

    • ✅ Los registros cruzados se guardan en fact_ingresos_davox con mes FISC, Id Servicio e Id Configuración Servicio.

    • ✅ Resumen: registros totales, activos cruzados, sin cruce con alerta en log.

    4.2 — Verificar Prerequisitos para Prefacturas (RN-29)¶

    Pasos de prueba¶

    1. Ir a la opción "Generación de Prefacturas".

    2. Verificar la matriz de prerequisitos:

    Componente Último mes liquidado
    Prorrateo Novedades Mes ejecutado
    Mantenimientos Mes cargado
    Indisponibilidades conciliadas Mes con consolidado generado

    Resultado esperado¶

    • ✅ Si los tres corresponden al mismo mes → botón "Generar Prefacturas" habilitado sin alerta.

    • ✅ Si alguno no coincide → el sistema genera alerta y solicita doble confirmación antes de continuar (RN-29).

    4.3 — Generación de Prefacturas¶

    Pasos de prueba¶

    1. Con prerequisitos del mes validados, hacer clic en "Generar Prefacturas" y confirmar.

    Resultado esperado¶

    • ✅ Se genera un registro en fact_prefactura por cada servicio activo del mes.

    • ✅ Novedad Recurrente: cruza con fact_indisponibilidad_mensual_conciliada → incluye descuento.

    • ✅ Cruza con fact_mantenimiento → incluye mantenimientos.

    • ✅ IVA = Total_menos_descuento_indisp × indicador_iva (RN-30).

    • ✅ moneda_mrc = 'USD' → MRC_COP = MRC × TRM del mes (RN-28).

    • ✅ Resumen: proveedores procesados, servicios prefacturados, valor total COP.

    4.4 — Validación Pagos vs Ingresos¶

    Pasos de prueba¶

    1. Ir a "Validación Pagos vs Ingresos".

    2. Seleccionar el mes y ejecutar la validación.

    Resultado esperado¶

    • ✅ El sistema cruza la prefactura generada con los ingresos Davox del mismo mes.

    • ✅ Se identifican varianzas entre valor prefacturado e ingresos reales.

    • ✅ Resultados guardados en fact_validacion_pagos_ingresos y visibles en pantalla.

    • ✅ El resultado es exportable.

    Flujo End-to-End Completo — Fase 1¶

    1. [Admin/Líder] Administración → Crear usuarios y asignar perfiles con permisos
         ↓

2. [Admin] Configuración → Cargar tablas paramétricas:
   Dim Proveedores, Dim Clientes, Dim Contratos, Dim TRM (1 por mes),
   Dim Ciudades, Dim Medios, Dim Configuración Servicios
         ↓

3. [Analista Conciliación] Indisponibilidades → "Cargar archivo base de incidencias"
   URL: /novedades/indisponibilidades
   Acción: cargar Closed_TERCEROS_Preview_<mes>.csv → procesar
         ↓

4. [Analista Conciliación] Indisponibilidades → "Validación ANS y exclusiones"
   Acción: ejecutar validación → verificar % disponibilidad y exclusiones
         ↓

5. [Analista Conciliación] Indisponibilidades → "Generación de archivos de conciliación"
   Acción: generar por proveedor → enviar al proveedor
         ↓

6. [Analista Conciliación] Indisponibilidades → "Cargue de respuestas de proveedores"
   Acción: cargar respuesta del proveedor
         ↓

7. [Analista Conciliación] Indisponibilidades → "Generación de consolidado mensual"
   Acción: ejecutar → consolidado disponible para Prefacturación
         ↓

8. [Analista Carriers] Prefacturación → "Cargar Ingresos Davox"
   URL: /novedades/prefacturacion
   Acción: cargar FISC_HIST_SC_<mes>.zip → procesar
         ↓

9. [Analista Carriers] Prefacturación → "Generación de Prefacturas"
   Acción: verificar prerequisitos → confirmar → generar
         ↓

10. [Analista Carriers] Prefacturación → "Validación Pagos vs Ingresos"
    Acción: ejecutar → exportar resultado
         ↓
✅ Ciclo mensual de Fase 1 completado
