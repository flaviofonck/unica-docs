    TASK-HU01: HU-01 — Cotización al Proveedor¶

    Estado: ⬜ Pendiente — scaffolding V7/V12/V13 existente
    Prioridad: 🔴 CRÍTICO — alcance contractual S4
    Referencia HU: docs/files/HU-01_Cotizacion.md

    Contexto¶

    Módulo para gestionar solicitudes de cotización a proveedores de telecomunicaciones. Permite registrar la solicitud, adjuntar especificaciones técnicas, recibir la respuesta del proveedor y registrar la decisión final (aprobado / rechazado / desierto).

    ⚠️ Las migraciones V7 (hu01 cotizacion), V12 y V13 ya existen. Verificar entidades y controlador antes de implementar para no duplicar artefactos.

    Tablas HU-01 (según docs/db_doc.md)¶

    Tabla Descripción
    dim_viabilidad_cotizacion Registro de solicitud de cotización con datos técnicos
    dim_cotizacion_proveedor Respuesta del proveedor a una viabilidad

    Backend¶

    Entidades JPA (verificar si ya existen en com.unica.app.entity):

    • ViabilidadCotizacion — idViabilidad, idConfiguracionServicio (FK), proveedor, descripcion, bwRequerido, ciudadOrigen, ciudadDestino, fechaSolicitud, estado, responsable

    • CotizacionProveedor — idCotizacion, idViabilidad (FK), proveedor, precioMensual, moneda, tiempoImplementacion, observaciones, fechaRespuesta, decision (APROBADA/RECHAZADA/DESIERTA)

    Servicios (CotizacionService):

    • crearViabilidad(dto) — registra solicitud con estado PENDIENTE

    • registrarRespuesta(idViabilidad, dto) — guarda respuesta del proveedor

    • aprobarCotizacion(idCotizacion) — cambia estado a APROBADA, registra en inventario

    • rechazarCotizacion(idCotizacion, motivo) — cambia estado a RECHAZADA

    • declararDesierta(idViabilidad, motivo) — sin respuesta válida de proveedor

    • listarViabilidades(filtros) — con filtros por estado, proveedor y fecha

    • exportarDetalle(idViabilidad) — genera CSV con especificación y respuesta

    Controlador REST (/api/viabilidad-cotizacion y /api/cotizacion-proveedor):

    Método Endpoint Descripción
    POST /api/viabilidad-cotizacion Crear nueva solicitud
    GET /api/viabilidad-cotizacion Listar con filtros (estado, proveedor, fechaDesde)
    GET /api/viabilidad-cotizacion/{id} Detalle de solicitud
    PUT /api/viabilidad-cotizacion/{id}/estado Actualizar estado de la viabilidad
    POST /api/cotizacion-proveedor Registrar respuesta del proveedor
    GET /api/cotizacion-proveedor/{idViabilidad} Ver cotizaciones de una viabilidad
    PUT /api/cotizacion-proveedor/{id}/decision Aprobar / rechazar / declarar desierta
    GET /api/viabilidad-cotizacion/{id}/exportar Exportar detalle en CSV

    DTOs: 

    // Request
record ViabilidadCotizacionRequest(
    Long idConfiguracionServicio, String proveedor, String descripcion,
    Double bwRequerido, String ciudadOrigen, String ciudadDestino, String responsable)

record CotizacionProveedorRequest(
    Long idViabilidad, String proveedor, Double precioMensual, String moneda,
    Integer tiempoImplementacion, String observaciones)

record DecisionCotizacionRequest(String decision, String motivo)

// Response
record ViabilidadCotizacionResponse(
    Long id, String proveedor, String estado, Double bwRequerido,
    String ciudadOrigen, String ciudadDestino, LocalDate fechaSolicitud,
    List<CotizacionProveedorResponse> cotizaciones)

    Frontend¶

    Componente: unica-frontend/src/app/components/cotizacion/

    Verificar si el directorio ya existe antes de crearlo.

    Pantallas:

    1. Lista de viabilidades — tabla con filtros de estado y proveedor; botón "Nueva viabilidad"

    2. Formulario nueva viabilidad — campos: proveedor, BW, ciudades, descripción, responsable
