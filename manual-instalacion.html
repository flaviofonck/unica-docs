    🖥️ Requisitos del Sistema¶

    Hardware Mínimo Recomendado¶

    • Procesador: Intel Core i5 o AMD Ryzen 5 (4 núcleos)
    • RAM: 8 GB (16 GB recomendado)
    • Disco: 20 GB libres (SSD recomendado)
    • Sistema Operativo: Windows 10/11 Professional (64-bit) o Windows Server 2019+

    Conectividad¶

    • Acceso a Internet para descarga de dependencias
    • Acceso al repositorio Git de Telefónica (credenciales proporcionadas)
    • Puertos disponibles: 8080 (backend), 4200 (frontend), 3306 (MySQL)

    📦 Software Requerido¶

    Versiones Exactas (Crítico - No usar otras versiones)¶

    Software Versión Exacta Propósito
    Java JDK 21.0.10 o superior (Eclipse Adoptium) Ejecución del backend Spring Boot
    MySQL Server 8.4.x Base de datos principal
    Node.js 20.x LTS (20.17.x recomendado) Runtime para Angular
    npm 11.9.0 o superior Gestor de paquetes frontend
    Angular CLI 21.2.2 Framework frontend
    Maven Incluido en el proyecto (wrapper) Build del backend
    Git 2.40+ Control de versiones y clonación del repositorio
    Cloudflare Tunnel (cloudflared) 2024.12.x o superior Acceso remoto seguro (opcional pero recomendado)

    🔧 Instalación de Dependencias¶

    1. Instalación de Java JDK 21¶

    Paso 1: Descargar JDK¶

    1. Visitar: https://adoptium.net/temurin/releases/
    2. Seleccionar:
    3. Version: 21 (LTS)
    4. Operating System: Windows
    5. Architecture: x64
    6. Descargar el instalador .msi

    Paso 2: Instalar JDK¶

    1. Ejecutar el instalador descargado
    2. Durante la instalación:
    3. ✅ Marcar "Set JAVA_HOME variable"
    4. ✅ Marcar "Add to PATH"
    5. Ruta sugerida: C:\Program Files\Eclipse Adoptium\jdk-21.x.x.x-hotspot
    6. Completar la instalación

    Paso 3: Verificar Instalación¶

    Abrir PowerShell o Command Prompt y ejecutar: 

    java -version
    Salida esperada: 
    openjdk version "21.0.10" 2024-01-16 LTS
OpenJDK Runtime Environment Temurin-21.0.10+9 (build 21.0.10+9-LTS)
OpenJDK 64-Bit Server VM Temurin-21.0.10+9 (build 21.0.10+9-LTS, mixed mode, sharing)

    Paso 4: Configurar JAVA_HOME (si no se configuró automáticamente)¶

    1. Abrir "Variables de entorno del sistema"
    2. En "Variables del sistema" → Nuevo
    3. Nombre: JAVA_HOME
    4. Valor: C:\Program Files\Eclipse Adoptium\jdk-21.0.10.7-hotspot (ajustar según la ruta real)
    5. Editar la variable PATH → Nuevo → %JAVA_HOME%\bin
    6. Reiniciar PowerShell y volver a verificar java -version

    2. Instalación de MySQL 8.4¶

    Paso 1: Descargar MySQL¶

    1. Visitar: https://dev.mysql.com/downloads/mysql/
    2. Seleccionar "MySQL Community Server 8.4.x" para Windows
    3. Descargar el instalador MSI (mysql-installer-community-8.4.x.msi)

    Paso 2: Instalar MySQL¶

    1. Ejecutar el instalador
    2. Seleccionar "Developer Default" o "Server only"
    3. En "Type and Networking":
    4. Config Type: Development Computer
    5. Port: 3306 (por defecto)
    6. ✅ Habilitar TCP/IP
    7. En "Authentication Method":
    8. Seleccionar: Use Strong Password Encryption (recomendado)
    9. En "Accounts and Roles":
    10. Root Password: admin (o personalizado según políticas de seguridad)
    11. ⚠️ IMPORTANTE: Anotar esta contraseña, se usará en la configuración
    12. En "Windows Service":
    13. ✅ Configure MySQL Server as a Windows Service
    14. Service Name: MySQL84
    15. ✅ Start the MySQL Server at System Startup
    16. Completar la instalación

    Paso 3: Verificar Instalación¶

    Abrir PowerShell y ejecutar: 

    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' --version
    Salida esperada: 
    mysql  Ver 8.4.x for Win64 on x86_64 (MySQL Community Server - GPL)

    Paso 4: Verificar Conexión¶

    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root -p
    - Ingresar la contraseña configurada (admin) - Si conecta correctamente, salir con exit;

    3. Instalación de Node.js y npm¶

    Paso 1: Descargar Node.js¶

    1. Visitar: https://nodejs.org/
    2. Descargar Node.js 20.x LTS (Long Term Support)
    3. Archivo: node-v20.x.x-x64.msi

    Paso 2: Instalar Node.js¶

    1. Ejecutar el instalador
    2. Durante la instalación:
    3. ✅ Aceptar todos los componentes por defecto
    4. ✅ Marcar "Automatically install the necessary tools" si aparece
    5. Completar la instalación

    Paso 3: Verificar Instalación¶

    Abrir PowerShell NUEVA (cerrar y abrir para recargar PATH) y ejecutar: 

    node --version
npm --version
    Salida esperada: 
    v20.17.x
11.9.x

    Paso 4: Actualizar npm (si es necesario)¶

    Si la versión de npm es inferior a 11.9.0: 

    npm install -g npm@11.9.0

    4. Instalación de Angular CLI¶

    Abrir PowerShell y ejecutar: 

    npm install -g @angular/cli@21.2.2

    Verificar instalación: 

    ng version
    Salida esperada: 
    Angular CLI: 21.2.2
Node: 20.x.x
Package Manager: npm 11.9.x

    5. Instalación de Git¶

    Paso 1: Descargar Git¶

    1. Visitar: https://git-scm.com/download/win
    2. Descargar el instalador de 64-bit Git for Windows Setup

    Paso 2: Instalar Git¶

    1. Ejecutar el instalador
    2. Durante la instalación, aceptar las opciones por defecto:
    3. ✅ Git Bash Here
    4. ✅ Git GUI Here
    5. Editor por defecto: Use Visual Studio Code as Git's default editor (o el preferido)
    6. PATH: Git from the command line and also from 3rd-party software
    7. SSH: Use bundled OpenSSH
    8. HTTPS: Use the OpenSSL library
    9. Line endings: Checkout Windows-style, commit Unix-style line endings
    10. Completar la instalación

    Paso 3: Verificar Instalación¶

    Abrir PowerShell NUEVA y ejecutar: 

    git --version
    Salida esperada: 
    git version 2.40.x.windows.x

    Paso 4: Configurar Git (Primera vez)¶

    Configurar nombre y email para commits: 

    git config --global user.name "Su Nombre"
git config --global user.email "su.email@telefonica.com"

    Verificar configuración: 

    git config --list

    🗄️ Configuración de Base de Datos¶

    1. Crear Base de Datos¶

    Abrir PowerShell y ejecutar: 

    # Conectar a MySQL
$env:MYSQL_PWD='admin'
& 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root -e "CREATE DATABASE IF NOT EXISTS unica_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

    2. Verificar Creación¶

    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root --password=admin -e "SHOW DATABASES LIKE 'unica_db';"
    Salida esperada: 
    +--------------------+
| Database (unica_db)|
+--------------------+
| unica_db           |
+--------------------+

    3. Configuración de Charset (Importante)¶

    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root --password=admin unica_db -e "ALTER DATABASE unica_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

    ⚠️ Nota sobre Migraciones¶

    Las tablas se crean automáticamente al arrancar el backend por primera vez mediante Flyway. No es necesario ejecutar scripts SQL manualmente.

    ⚙️ Configuración de la Aplicación¶

    1. Crear Cuenta de GitHub y Obtener Acceso¶

    Paso 1: Crear Cuenta en GitHub (si no tiene una)¶

    1. Visitar: https://github.com/signup
    2. Ingresar:
    3. Email: Usar el correo corporativo de Telefónica
    4. Contraseña: Crear una contraseña segura
    5. Username: Elegir un nombre de usuario (ej: juanperez-telefonica)
    6. Verificar el email (GitHub enviará un código de verificación)
    7. Completar el perfil básico

    Paso 2: Solicitar Permisos de Colaborador¶

    ⚠️ IMPORTANTE: El repositorio es privado. Debe solicitar acceso siguiendo estos pasos:

    1. Enviar su usuario de GitHub al administrador del proyecto
    2. Ejemplo: Si su usuario es juanperez-telefonica, enviar ese nombre

    3. El administrador agregará su usuario como colaborador del repositorio

    4. Esperar confirmación de acceso

    5. Recibirá un email de GitHub con la invitación

    6. Título: "You've been invited to collaborate on..."

    7. Aceptar la invitación

    8. Hacer clic en el link del email o visitar: https://github.com/flaviofonck/unica-telefonica-produccion

    9. Hacer clic en "Accept invitation"

    10. Verificar acceso

    11. Una vez aceptada la invitación, podrá ver el repositorio en: https://github.com/flaviofonck/unica-telefonica-produccion

    2. Clonar el Repositorio¶

    URL del Repositorio¶

    https://github.com/flaviofonck/unica-telefonica-produccion

    Clonar vía HTTPS (Recomendado)¶

    Abrir PowerShell y ejecutar: 

    # Crear carpeta contenedora (opcional)
mkdir C:\proyectos
cd C:\proyectos

# Clonar el repositorio
git clone https://github.com/flaviofonck/unica-telefonica-produccion.git unica

    Comando completo en una línea: 

    git clone https://github.com/flaviofonck/unica-telefonica-produccion.git unica

    Autenticación con GitHub¶

    Cuando ejecute git clone, GitHub solicitará autenticación:

    Método 1: GitHub Desktop (Más Fácil - Recomendado para Windows)

    1. Descargar GitHub Desktop: https://desktop.github.com/
    2. Instalar y hacer login con su cuenta de GitHub
    3. GitHub Desktop maneja automáticamente la autenticación
    4. Usar File → Clone Repository para clonar gráficamente

    Método 2: Personal Access Token (PAT) - Línea de Comandos

    GitHub ya no acepta contraseñas en línea de comandos. Debe usar un Personal Access Token:

    1. Generar un Token:
    2. Visitar: https://github.com/settings/tokens
    3. Hacer clic en "Generate new token" → "Generate new token (classic)"
    4. Note: unica-telefonica-token
    5. Expiration: 90 días (o según políticas de seguridad)
    6. Scopes: Marcar ✅ repo (acceso completo a repositorios privados)
    7. Hacer clic en "Generate token"

    8. ⚠️ COPIAR EL TOKEN (solo se muestra una vez): ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

    9. Usar el Token al Clonar: 

      git clone https://github.com/flaviofonck/unica-telefonica-produccion.git unica

    10. Cuando pida username: Ingresar su usuario de GitHub

    11. Cuando pida password: Pegar el token (no la contraseña de GitHub)

    12. Guardar Credenciales (para no ingresarlas cada vez): 

      git config --global credential.helper manager

    13. Windows abrirá el Credential Manager y guardará el token automáticamente

    Método 3: SSH Keys (Avanzado)

    Si prefiere usar SSH: 

    git clone git@github.com:flaviofonck/unica-telefonica-produccion.git unica
    - Requiere configurar claves SSH en GitHub: https://docs.github.com/en/authentication/connecting-to-github-with-ssh

    Resultado Esperado¶

    Después de clonar, se creará la carpeta unica\ con toda la estructura del proyecto: 

    C:\proyectos\unica\
├── src/
├── unica-frontend/
├── docs/
├── backend.bat
├── frontend.bat
├── pom.xml
└── ...

    ⚠️ IMPORTANTE: El proyecto debe clonarse en una ruta sin espacios ni caracteres especiales.

    Rutas válidas: - ✅ C:\unica\ - ✅ C:\proyectos\unica\ - ✅ D:\apps\unica\

    Rutas NO válidas: - ❌ C:\Program Files\unica\ (tiene espacios) - ❌ C:\Mis Documentos\unica\ (tiene espacios) - ❌ C:\carpeta con espacios\unica\

    3. Configurar Credenciales de Base de Datos¶

    Editar el archivo: 

    C:\unica\src\main\resources\application.properties

    Verificar las siguientes líneas (ajustar si se cambió la contraseña de MySQL): 

    spring.datasource.url=jdbc:mysql://localhost:3306/unica_db?serverTimezone=UTC&useUnicode=true&characterEncoding=UTF-8&connectionCollation=utf8mb4_unicode_ci&rewriteBatchedStatements=true
spring.datasource.username=root
spring.datasource.password=admin

    Si se cambió la contraseña de MySQL, modificar la línea: 

    spring.datasource.password=SU_CONTRASEÑA_REAL

    4. Configurar Variables de Entorno (Opcional - Scripts incluidos)¶

    Los scripts backend.bat y frontend.bat incluidos ya configuran automáticamente JAVA_HOME. Si se desea ejecutar manualmente los comandos Maven, verificar que JAVA_HOME esté configurado:

    $env:JAVA_HOME
    Debe mostrar algo como: 
    C:\Program Files\Eclipse Adoptium\jdk-21.0.10.7-hotspot

    🚀 Ejecución de la Aplicación¶

    Opción 1: Scripts Automáticos (Recomendado)¶

    Iniciar Backend¶

    En PowerShell, dentro de la carpeta del proyecto: 

    cd C:\unica
.\backend.bat

    Proceso: 1. Verifica y libera el puerto 8080 2. Configura JAVA_HOME automáticamente 3. Ejecuta Maven Wrapper 4. Aplica migraciones Flyway (primera vez crea todas las tablas) 5. Inicia Spring Boot

    Esperar a ver: 

    Started AppApplication in X.XXX seconds

    ⚠️ No cerrar esta ventana - El backend debe permanecer en ejecución.

    Iniciar Frontend¶

    En otra ventana de PowerShell: 

    cd C:\unica
.\frontend.bat

    Proceso: 1. Instala dependencias npm (primera vez, puede tardar varios minutos) 2. Inicia servidor Angular de desarrollo 3. Compila la aplicación

    Esperar a ver: 

    ✔ Browser application bundle generation complete.
** Angular Live Development Server is listening on localhost:4200 **

    Opción 2: Comandos Manuales¶

    Backend Manual¶

    🌐 Configuración de Cloudflare Tunnel (Acceso Remoto)¶

    ⚠️ IMPORTANTE: Esta sección es necesaria para permitir el acceso remoto a la aplicación ÚNICA a través de Internet de forma segura, sin necesidad de abrir puertos en el firewall ni configurar port forwarding.

    ¿Qué es Cloudflare Tunnel?¶

    Cloudflare Tunnel crea una conexión segura entre su servidor local y la red de Cloudflare, permitiendo que la aplicación sea accesible desde Internet mediante URLs públicas (ej: https://unica.fonck.app).

    Ventajas: - ✅ Sin necesidad de IP pública estática - ✅ Sin configuración de router/firewall - ✅ Certificados SSL automáticos (HTTPS) - ✅ Protección DDoS de Cloudflare - ✅ Acceso controlado por el proveedor

    1. Instalar Cloudflared¶

    Paso 1: Descargar el Ejecutable¶

    1. Visitar: https://github.com/cloudflare/cloudflared/releases
    2. Descargar la versión para Windows (64-bit):
    3. Buscar el archivo: cloudflared-windows-amd64.exe
    4. Última versión estable (ej: 2024.12.2)

    Paso 2: Instalar en el Sistema¶

    # Crear carpeta de instalación
New-Item -ItemType Directory -Force -Path "C:\Program Files (x86)\cloudflared"

# Mover el ejecutable descargado (ajustar ruta de Downloads)
Move-Item "$env:USERPROFILE\Downloads\cloudflared-windows-amd64.exe" "C:\Program Files (x86)\cloudflared\cloudflared.exe"

# Verificar instalación
& 'C:\Program Files (x86)\cloudflared\cloudflared.exe' --version

    Resultado esperado: 

    cloudflared version 2024.12.2 (built YYYY-MM-DD)

    2. Preparar Configuración del Túnel¶

    ⚠️ PASO COORDINADO CON EL PROVEEDOR:

    El proveedor del sistema (QCS/desarrollador) le proporcionará un archivo de credenciales que debe colocar en su máquina. Este archivo es único por instalación y contiene las claves de autenticación del túnel.

    Paso 1: Crear Carpeta de Credenciales¶

    # Crear carpeta .cloudflared en el perfil de usuario
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.cloudflared"

    Paso 2: Recibir Archivo de Credenciales¶

    El proveedor le enviará un archivo con nombre similar a: 

    f123a480-a795-4497-8972-eee3474cad45.json

    Debe colocar este archivo en: 

    C:\Users\<SU_USUARIO>\.cloudflared\<UUID>.json

    Ejemplo: 

    C:\Users\admin.telefonica\.cloudflared\f123a480-a795-4497-8972-eee3474cad45.json

    ⚠️ NO compartir ni modificar este archivo - contiene credenciales de autenticación.

    3. Actualizar Rutas en Configuración¶

    El archivo cloudflared-config.yml en la raíz del proyecto contiene las rutas de configuración. Debe actualizar la línea de credentials-file con su usuario de Windows.

    Ubicación del archivo: 

    C:\unica\cloudflared-config.yml

    Editar con Bloc de Notas:

    # cloudflared-config.yml — ÚNICA App
tunnel: f123a480-a795-4497-8972-eee3474cad45
credentials-file: C:\Users\<SU_USUARIO>\.cloudflared\f123a480-a795-4497-8972-eee3474cad45.json

ingress:
  - hostname: unica.fonck.app
    service: http://localhost:4200
  - hostname: api-unica.fonck.app
    service: http://localhost:8080
  - service: http_status:404

    Reemplazar: - <SU_USUARIO> → Su nombre de usuario de Windows (ej: admin.telefonica) - El UUID del tunnel (números largos) debe coincidir con el nombre del archivo JSON

    4. Ejecutar el Túnel¶

    🛠️ Solución de Problemas¶

    Problema: "El término 'java' no se reconoce"¶

    Causa: JAVA_HOME no está configurado correctamente.

    Solución: 1. Verificar instalación de JDK: dir "C:\Program Files\Eclipse Adoptium\" 2. Configurar JAVA_HOME en Variables de Entorno del Sistema 3. Reiniciar PowerShell

    Problema: "Cannot connect to MySQL Server"¶

    Causa: MySQL no está en ejecución o las credenciales son incorrectas.

    Solución: 1. Verificar que el servicio de MySQL esté corriendo: 

    Get-Service -Name MySQL84
    Si está detenido: 
    Start-Service -Name MySQL84
    2. Verificar credenciales en application.properties 3. Probar conexión manual: 
    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root -p

    Problema: "Port 8080 already in use"¶

    Causa: Otro proceso está usando el puerto 8080.

    Solución: 1. Identificar el proceso: 

    netstat -ano | findstr :8080
    2. Detener el proceso (anotar el PID del resultado anterior): 
    Stop-Process -Id <PID> -Force
    3. Reintentar .\backend.bat

    Problema: "npm ERR! ENOENT"¶

    Causa: node_modules corrupto o npm cache con problemas.

    Solución: 

    cd C:\unica\unica-frontend
Remove-Item -Recurse -Force node_modules
npm cache clean --force
npm install

    Problema: "Flyway migration failed"¶

    Causa: Una migración ya fue ejecutada parcialmente y quedó marcada como fallida.

    Solución: 

    # Ver el historial de migraciones
$env:MYSQL_PWD='admin'
& 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root unica_db -e "SELECT version, description, success FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 10;"

# Si hay una migración con success=0, eliminarla:
& 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root --password=admin unica_db -e "DELETE FROM flyway_schema_history WHERE version='X' AND success=0;"

# Reiniciar backend
.\backend.bat

    Problema: "Angular CLI not found"¶

    Causa: Angular CLI no está instalado globalmente o npm global no está en PATH.

    Solución: 

    npm list -g @angular/cli
# Si no aparece o la versión es incorrecta:
npm uninstall -g @angular/cli
npm install -g @angular/cli@21.2.2

    Problema: Caracteres extraños en la base de datos (mojibake)¶

    Causa: Problemas de encoding UTF-8.

    Solución: 1. Verificar charset de la base de datos: 

    & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root --password=admin unica_db -e "SHOW CREATE DATABASE unica_db;"
    Debe mostrar: CHARACTER SET utf8mb4

    1. Si está incorrecto, corregirlo: 

      & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysql.exe' -u root --password=admin unica_db -e "ALTER DATABASE unica_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

    2. Reiniciar el backend.

    Problema: "git: command not found" o "El término 'git' no se reconoce"¶

    Causa: Git no está instalado o no está en el PATH.

    Solución: 1. Verificar si Git está instalado: 

    dir "C:\Program Files\Git"
    2. Si no está instalado, seguir la sección "5. Instalación de Git" 3. Si está instalado pero no se reconoce el comando: - Agregar C:\Program Files\Git\cmd a la variable PATH del sistema - Reiniciar PowerShell

    Problema: "Authentication failed" al clonar el repositorio¶

    Causa: Credenciales incorrectas o token de acceso inválido.

    Solución: 1. Verificar que las credenciales proporcionadas sean correctas 2. Si se usa token de acceso personal (PAT): - Verificar que el token no haya expirado - Verificar que el token tenga permisos de lectura del repositorio 3. Limpiar credenciales guardadas e intentar de nuevo: 

    git credential-manager clear
    4. Intentar clonar nuevamente, ingresando credenciales correctas

    Problema: "fatal: destination path 'unica' already exists"¶

    Causa: Ya existe una carpeta unica en la ubicación donde se intenta clonar.

    Solución: 1. Opción A - Clonar en otra ubicación: 

    git clone <URL> unica-nuevo
    2. Opción B - Eliminar la carpeta existente (si no contiene cambios importantes): 
    Remove-Item -Recurse -Force unica
git clone <URL> unica

    Problema: "SSL certificate problem" al clonar¶

    Causa: Problemas con certificados SSL corporativos o proxy.

    Solución: 1. Temporal (solo para probar): 

    $env:GIT_SSL_NO_VERIFY = 'true'
git clone <URL>
    2. Permanente (recomendado para entornos corporativos): - Obtener el certificado corporativo (.crt) - Configurar Git para usarlo: 
    git config --global http.sslCAInfo "C:\ruta\al\certificado.crt"
    3. Si hay un proxy corporativo, configurarlo: 
    git config --global http.proxy http://proxy.empresa.com:puerto

    Problema: Túnel no se conecta - "failed to authenticate"¶

    Causa: Archivo de credenciales JSON incorrecto, inexistente o con permisos incorrectos.

    Solución: 1. Verificar que el archivo JSON existe: 

    Test-Path "$env:USERPROFILE\.cloudflared\*.json"
    2. Verificar que la ruta en cloudflared-config.yml es correcta y coincide con su usuario de Windows 3. Verificar que el UUID en cloudflared-config.yml coincide con el nombre del archivo JSON 4. Si el archivo es nuevo, asegurar que fue copiado completamente (no truncado) 5. Solicitar al proveedor un nuevo archivo de credenciales si el problema persiste

    Problema: "tunnel connections not registered" en tunnel.bat¶

    Causa: Backend o frontend no están corriendo, o están en puertos incorrectos.

    Solución: 1. Verificar orden de inicio: - Primero: backend.bat (esperar "Started AppApplication") - Segundo: frontend.bat (esperar "listening on localhost:4200") - Tercero: tunnel.bat 2. Verificar que los puertos están activos: 

    # Backend en 8080
Test-NetConnection -ComputerName localhost -Port 8080
# Frontend en 4200
Test-NetConnection -ComputerName localhost -Port 4200
    3. Si algún puerto no responde, detener el túnel y reiniciar backend/frontend primero

    Problema: "cloudflared.exe not found" al ejecutar tunnel.bat¶

    Causa: Cloudflared no está instalado en la ruta esperada.

    Solución: 1. Verificar instalación: 

    Test-Path "C:\Program Files (x86)\cloudflared\cloudflared.exe"
    2. Si no existe, seguir pasos de instalación en Sección 7 3. Si está en otra ubicación, actualizar la variable PATH en tunnel.bat: 
    set "PATH=%PATH%;<ruta_donde_esta_cloudflared>"

    Problema: Túnel conecta pero no se puede acceder desde Internet¶

    Causa: Configuración DNS no completada o dominios no apuntados al túnel.

    Solución: 1. ⚠️ Esto es responsabilidad del proveedor - no puede resolverse localmente 2. Verificar que el túnel está corriendo correctamente (debe mostrar "Connection established") 3. Coordinar sesión remota con el proveedor para: - Configurar registros DNS en Cloudflare - Verificar que los dominios estén activos - Probar acceso desde Internet 4. Mientras tanto, verificar acceso local en http://localhost:4200 y http://localhost:8080

    Problema: "ERR_TOO_MANY_REDIRECTS" al acceder a URL pública del túnel¶

    Causa: Configuración de ingress incorrecta en cloudflared-config.yml.

    Solución: 1. Verificar que el archivo cloudflared-config.yml tiene la estructura correcta: 

    ingress:
  - hostname: unica.fonck.app
    service: http://localhost:4200
  - hostname: api-unica.fonck.app
    service: http://localhost:8080
  - service: http_status:404
    2. No debe haber líneas duplicadas ni tabulaciones (usar solo espacios) 3. Reiniciar el túnel después de cualquier cambio en este archivo

      & 'C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqldump.exe' -u root --password=admin unica_db > backup_$(Get-Date -Format 'yyyyMMdd').sql
    4. Pruebas: Después de actualizar, probar las funcionalidades críticas antes de volver a operar normalmente
    5. Documentación: Leer las notas de la versión (release notes) para conocer los cambios
