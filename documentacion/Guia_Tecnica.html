    Campos críticos de DIM_CONFIGURACION_SERVICIOS¶

    Esta tabla tiene 50 campos definidos en el modelo de datos oficial (ver Documento 04 — Diccionario de Datos). Los campos más relevantes para los cruces entre módulos son:

    Campo DB Tipo Usado por Regla
    id BIGINT HU-07, HU-08 (FK) Llave interna de cruce
    id_servicio VARCHAR(50) HU-06, HU-07, HU-08 (FK) PK de negocio — del reporte instalaciones
    ds VARCHAR(50) HU-07, HU-04 (llave cruce) DS único — no puede tener otra config activa
    nit_proveedor VARCHAR(15) HU-07 FK → dim_proveedores.nit
    proveedor VARCHAR(200) HU-07 Nombre del proveedor
    nit_cliente VARCHAR(10) HU-08 FK → dim_clientes.nit
    codigo_dane_ciudad VARCHAR(10) HU-01 (cobertura) FK → dim_ciudades.codigo_dane
    fecha_baja DATE HU-07 Validación: Incident_Reported_Date > fecha_baja
    estado_configuracion VARCHAR(30) HU-04 ACTIVO | BAJA
    estado_baja VARCHAR(50) HU-04 BAJA POR MIGRACIÓN / BAJA POR CLIENTE / BAJA POR ASEGURAMIENTO / SUSPENSIÓN POR CARTERA
    condicion_facturacion VARCHAR(80) HU-06 Mensual Vencido / Anticipado / Trimestral — autocompleta desde dim_contratos
    fecha_fin_clausula DATE HU-04 Terminación anticipada
    mrc DECIMAL(15,2) HU-04, HU-06 Cargo mensual recurrente
    moneda_mrc VARCHAR(3) HU-06 COP | USD — aplica conversión TRM
    fecha_tope_pago_baja DATE HU-04 Fecha notificación + 30 días (configurable)
    pago_proyectado_preaviso DECIMAL(15,2) HU-04 (MRC/30) × días_preaviso
    pago_proyectado_terminacion DECIMAL(15,2) HU-04 Calculado según condiciones contrato
    numero_contrato VARCHAR(50) HU-05, HU-06 FK → dim_contratos. Trigger: propaga condicion_pago y condicion_facturacion

    Reglas de negocio críticas¶

    • Flujo de aprobación del líder de área para toda edición de tabla paramétrica.

    • Un único valor de TRM por mes (restricción de integridad de datos).

    • Cambios en DIM_CONTRATOS (condiciones pago/facturación) → propagar automáticamente a DIM_CONFIGURACION_SERVICIOS.

1. Verificar que no exista registro en control_ejecucion_prorrateo para mesLiquidacion (idempotencia)
2. Para cada Id Servicio en DIM_SERVICIOS:
   a. Identificar novedades asociadas en DIM_CONFIGURACION_SERVICIOS
   b. Según condicion_facturacion:
      - MENSUAL_VENCIDO → mes a facturar = mes anterior al de liquidación
      - MENSUAL_ANTICIPADO → mes a facturar = mes de liquidación

   c. Para cada tipo de novedad calcular valor proporcional en COP:
      valor_cop = (MRC × dias_activos_en_mes) / dias_totales_mes
      (convertir moneda usando DIM_TRM si aplica)
3. Para servicios sin novedades (recurrentes): generar registro con tipo_novedad = 'RECURRENTE'
4. Guardar todos los registros en FACT_PRORRATEO_NOVEDADES_SERVICIOS
5. Registrar ejecución en control_ejecucion_prorrateo

cargarArchivoIncidencias(MultipartFile, String mesReporte):
  1. Verificar que no exista carga para mesReporte en control_carga_incidencias
  2. Parsear archivo TXT/CSV (20k-30k registros) de forma ASÍNCRONA (@Async)
  3. Filtro 1 — cruzar Numero_DS con DIM_CONFIGURACION_SERVICIOS:
     - DS encontrada Y Incident_Reported_Date > Fecha_Baja → estado = "Registro coincidente DS"
  4. Filtro 2 — comparar Vendor_Ticket_Number con DIM_EXCLUSIONES_REMEDY:
     - Ver tabla de lógica de sobrescritura (sección 2.4 HU-07)
  5. Insertar en FACT_TICKETS_INDISPONIBILIDAD
  6. Registrar en control_carga_incidencias

calcularIndisponibilidades(String mesReporte):
  Para cada registro con estado "Registro coincidente DS" o "Registro identificado Remedy-Vendor":
  1. indisp_mins = TIMESTAMPDIFF(MINUTE, fecha_apertura, fecha_cierre)
  2. dias_mes = días del mes de mesReporte
  3. pct_disponibilidad = (((dias_mes * 24 * 60) - indisp_mins) / (dias_mes * 24 * 60)) * 100
  4. pct_descuento = buscar en DIM_ANS_PROVEEDORES por proveedor donde pct_disponibilidad esté en rango
  5. fallo_responsabilidad = (observaciones IN DIM_MOTIVOS_FALLAS) ? true : false
  6. pct_final_descuento = fallo_responsabilidad ? 0 : pct_descuento
  7. Guardar en FACT_INDISPONIBILIDAD_MENSUAL

generarArchivoConciliacion(Long idProveedor, String mesReporte):
  → Exportar Excel: Informe_de_indisponibilidades_NombreProveedor_MM_AAAA

cargarRespuestasProveedor(List<RespuestaDTO>):
  → Actualizar campos viabilidad_proveedor y observaciones_conciliacion en
    FACT_INDISPONIBILIDAD_MENSUAL_CONCILIADA
  → Calcular mes_facturacion según condicion_facturacion del servicio:
     - MENSUAL_VENCIDO → mes_reporte
     - MENSUAL_ANTICIPADO → mes_reporte + 1
     - TRIMESTRAL_ANTICIPADO → primer mes del siguiente trimestre

generarConsolidadoMensual(String mesReporte):
  → Exportar: Informe_de_indisponibilidades_NombreProveedor_MM_AAAA_Final.xlsx
  → Este archivo alimenta directamente HU-08 (FACT_INDISPONIBILIDAD_MENSUAL_CONCILIADA)

cargarIngresosDavox(MultipartFile zipFile, String mesInforme):
  1. Verificar que no exista carga para mesInforme en control_carga_fisc
  2. Descomprimir ZIP → extraer TXT (~700MB)
  3. Procesar ASÍNCRONAMENTE:
     a. Filtrar: ESTADO_CUS = "ACTIVO" (~700MB → ~50MB)
     b. Cruzar CUS con DIM_CONFIGURACION_SERVICIOS → obtener Id Servicio, Id Configuracion
     c. Insertar en FACT_INGRESOS_DAVOX
  4. Registrar en control_carga_fisc

verificarMatrizPrevia(String mesFacturacion):
  Retornar:
  - Último mes donde se corrió prorrateo (desde control_ejecucion_prorrateo)
  - Último mes donde se cargaron mantenimientos (desde fact_mantenimientos)
  - Último mes donde se generó consolidado de conciliación (desde fact_indisponibilidad_mensual_conciliada)
  - Flag: todos_meses_iguales (boolean)

generarPrefacturas(String mesFacturacion, boolean confirmarInconsistencia):
  1. Si verificarMatrizPrevia detecta inconsistencia y !confirmarInconsistencia → lanzar excepción
  2. Trasladar registros de FACT_PRORRATEO_NOVEDADES_SERVICIOS → FACT_PREFACTURA
  3. Para registros con novedad = 'Recurrente':
     a. Cruzar con FACT_INDISPONIBILIDAD_MENSUAL_CONCILIADA (llave: id_configuracion + mes)
        → obtener descuento_indisp
     b. Cruzar con FACT_MANTENIMIENTOS (mismas llaves)
        → obtener valor_mantenimiento
  4. Calcular:
     cobro_cop_1 = total - descuento_indisp
     cobro_iva = cobro_cop_1 × pct_iva
     cobro_cop_total = cobro_cop_1 + cobro_iva
  5. Estado inicial: 'Por Validar'

calcularResumenPrefactura(Long idProveedor, String mes):
  Solo sobre registros con estado_registro = 'Válido':
  - NUMERO_IDs por moneda y por novedad
  - COBRO_COP_1 = SUMA(total - descuento_indisp) agrupado
  - COBRO_IVA_19 = COBRO_COP_1 × % IVA
  - COBRO_COP = COBRO_COP_1 + COBRO_IVA_19
  - Total sin IVA, INDISP, IVA final, TOTAL FINAL

cerrarConsolidadoMes(String mes):
  → Todos los 'Por Validar' → 'Válido' (excepto explícitamente 'No Válido')
  → Exportar Excel de FACT_PREFACTURA filtrado por mes
